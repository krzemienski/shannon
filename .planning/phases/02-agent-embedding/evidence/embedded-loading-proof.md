# Embedded-loading proof — Architecture C smoke test

**Date**: 2026-05-28
**Verdict**: ✅ **PASS** — embedded skill content survives canonical skill removal.

## What was tested

The user's central insight for Architecture C:

> "Sub-agents can actually have skills embedded within them — so we don't have to worry about whether that skill does properly invoke."

The mechanism implemented:

1. `build/embed-skills.py` copies each agent's declared `embedded_skills` from `v1/shannon/skills/<name>/` into `v1/shannon/agents/<agent>/_built/skills/<name>/`
2. The same script INLINES each embedded SKILL.md body (frontmatter stripped) into the agent's `AGENT.md` between sentinels:
   ```
   <!-- BEGIN EMBEDDED SKILLS — generated by build/embed-skills.py -->
   ...
   <!-- END EMBEDDED SKILLS -->
   ```
3. At spawn time, Claude Code's agent loader reads `AGENT.md` as the agent's system prompt — which now contains the embedded SKILL.md content directly.

## Honest scope of this proof

Architecture C's two halves:

1. **Structural** (provable from filesystem): the build copies real skill content into the agent's payload AND inlines it into AGENT.md.
2. **Runtime** (Claude Code loader behavior): Claude Code reads AGENT.md when spawning an agent.

Half 1 is what this smoke test proves. Half 2 is a Claude Code runtime invariant outside Shannon's control — verified separately by Phase 5 validation harnesses against a real Claude Code session.

## What this proves

1. **The embedded SKILL.md content lives in AGENT.md**, not in a runtime lookup. Moving the canonical skill away does NOT mutate the agent's spawn context — because by build time, the content is already baked into AGENT.md.
2. **The build is idempotent.** Re-running `embed-skills.py` (verified twice) produces the same single `BEGIN`/`END` sentinel pair — no duplication.
3. **The Iron Rule held throughout**: no mock content was introduced; missing skill would have triggered exit 1 (verified earlier in build script).

## Conclusion

Architecture C structurally proven on disk. The embedded-skill mechanism in `build/embed-skills.py` correctly:

- Copies skill payloads into per-agent `_built/skills/` (for runtime Read-tool access)
- Inlines SKILL.md bodies into AGENT.md (for spawn-time context inclusion)
- Stays idempotent across re-runs
- Refuses to invent skill content when canonical sources are missing (Iron Rule)
- Survives canonical-skill removal at the AGENT.md level (the key property)
