### Task 2 ‚Äî For each kept agent, list embedded skills

This is the central Architecture C decision. Each agent's `manifest.yml` declares its embedded skills. The build script (`build/embed-skills.py` ‚Äî Plan 02-02) reads these manifests and copies the listed skills into `agents/<name>/_built/skills/` at build time.

**Constraints:**

1. **Every embedded skill MUST exist in `v1/shannon/skills/`** (the 31 curated set from Phase 1)
2. **A skill may be embedded in multiple agents** (duplication is acceptable per Architecture C)
3. **Minimum 1 embedded skill per agent** (an agent with zero skills isn't doing the embedding work)
4. **Maximum 6-8 embedded skills per agent** (above this, the agent's context budget on spawn gets crowded)

**Tentative embedding map (for user review):**


Wait ‚Äî the tentative map above has issues to flag for user input:

- **`meta-judge` embeds `judge`** ‚Äî but Architecture C also wants `judge` to be a standalone skill in `v1/shannon/skills/`. Both happen (standalone AND embedded), per Architecture C's hybrid model.
- **`critic` embeds `critique`** ‚Äî but `critique` was MERGED into `judge` in Phase 1; it's no longer a standalone skill. The `critic` agent should embed `judge` and use it in severity mode. Need to update the map.
- **`team-validator` and `validator` overlap heavily** ‚Äî Plan 02-01 should consider merging them into one agent OR clearly distinguishing their roles.

### Task 3 ‚Äî Cascade-conflict analysis

When two agents embed two skills that have same-named `references/foo.md` files, the build step would have a collision. Detect and document.

**Action:** For every pair of skills that might land in the same agent's `_built/skills/`, check whether their `references/` directories have name overlaps. Write findings to `evidence/cascade-conflicts.md`.

This is a pre-flight check for Plan 02-02's build script ‚Äî surfaces problems before code is written.

**Verify (gate):**
- Conflict scan ran over every agent's embedded_skills list
- `cascade-conflicts.md` exists (even if empty ‚Äî "no conflicts found" is also a result)
