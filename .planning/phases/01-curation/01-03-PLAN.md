### Task 2 — Validate copied set

Apply the same V7_FUNCTIONAL_VALIDATION criteria 2 + 3 to the copied set:

```bash
python3 <<'PY'
import os, yaml, re
SKILLS = '../../../skills'
pass_yaml = fail_yaml = 0
pass_cascade = fail_cascade = 0
for n in sorted(os.listdir(SKILLS)):
    p = f'{SKILLS}/{n}/SKILL.md'
    if not os.path.isfile(p): continue
    t = open(p).read()
    try:
        e = t.find('---', 3); fm = yaml.safe_load(t[3:e])
        assert fm.get('name') and fm.get('description'); pass_yaml += 1
    except: fail_yaml += 1
    # Cascade
    dir_ = f'{SKILLS}/{n}'
    for ref in re.findall(r'references/[a-z0-9_-]+\.md', open(p).read()):
        if os.path.isfile(f'{dir_}/{ref}'): pass_cascade += 1
        else: fail_cascade += 1
print(f'YAML: {pass_yaml}/{pass_yaml+fail_yaml} pass')
print(f'Cascade: {pass_cascade}/{pass_cascade+fail_cascade} pass')
import sys; sys.exit(0 if fail_yaml==0 and fail_cascade==0 else 1)
PY
```

**Verify (gate):**
- Script exits 0 (all YAML valid, all cascade references resolve)
- Output saved to `evidence/copied-skills-validation.log`

**Failure mode → fix:**
- If YAML fails for a copied skill, that's the same bug from the v7 audits (colon-in-description). Apply the same fix (replace `:` with `—` in the offending phrase).
- If cascade fails, the source skill in `shannon-framework/skills/<n>/references/` may be missing a file. Either copy the missing file from a different source OR remove the broken reference from the SKILL.md.
