### Task 2 — Map each of the 67 skills against the 5 pillars

The 5 pillars (from BRIEF.md):
1. **Embedded sub-agent skills**
2. **Orchestration**
3. **Iron Rule validation**
4. **Meta-judge consensus**
5. **Self-instrumented**

Plus a special category **"Foundational/Adjacent"** for skills that don't serve any pillar directly but are required infrastructure (e.g., `python-agent-sdk` enables the SDK harness; `commit-work` is dev tooling).

**Action:** Produce `evidence/categorization-matrix.md` with one Markdown table row per skill:

```markdown
| skill | pillars | source | cascade | role | notes |
|---|---|---|---|---|---|
| functional-validation | 3 (Iron Rule) | anthropic-skills | 4 | core | the Iron Rule entry point |
| meta-judge-rubric | 4 (Meta-judge) | shannon-original | 0 | core | rubric YAML generator — paired with `agents/meta-judge.md` |
| dispatch-parallel | 2 (Orchestration) | shannon-merge | 0 | core | single-message multi-Task enforcer |
| ... | ... | ... | ... | ... | ... |
```

The `pillars` column lists ALL pillars a skill serves (a skill can serve multiple). Use `0 (none)` for skills that don't directly serve any.

The `role` column is one of:
- `core` — directly delivers a pillar capability; high probability of KEEP
- `adjacent` — enables a pillar but isn't itself the capability; likely KEEP
- `infra` — required infrastructure (e.g., python-agent-sdk for the harness); likely KEEP
- `tangent` — useful elsewhere but not for v0.1.0 Shannon; likely CUT
- `duplicate` — overlapping with another skill; KEEP one, CUT others
- `historical` — v5/v6/v7 carryover not aligned with v1 BRIEF; likely CUT

**Verify (gate inside the task):**
- `evidence/categorization-matrix.md` has exactly 67 data rows (plus header + separator)
- Every row has all 6 columns populated (no empty cells)
- Every `pillars` entry references either "0 (none)" or one or more of: 1, 2, 3, 4, 5
- Every `role` entry is one of the 6 valid values

**Failure mode → fix:** if a skill resists categorization, do not skip it. Mark it as `role: tangent` with `notes: "category unclear — review in 01-02"`. The next sub-plan can revisit.
